Базовые пояснения.

При создании скилла:

- Начальный номер оператора - индекс с которого начинаемся именование оператора в формате Operator + index.
Т.о. если начальный номер оператора = 1 и количество операторов 10, то список операторов обудет выглядеть следующим образом:
Operator1
Operator2
Operator3
...
Operator11

- Интервал распределения вызовов это интервал в секундах, в течение которого будут поступать вызовы, которые указаны в таблице справа (распределение по длительности) случайным образом сгенерированные.

- Правила приоритизации - правила, по которым будет осуществляться та самаая приоритизация.
Формат правила следующий
if условие then priority = n
Условия:
queue in (SG_50,SG_21...) > 1 - очередь на одном из скиллов больше 1
queue in each (SG_50,SG_21...) > 1 - очередь на каждом скилле больше 1
timeWait > 180 - время ожидания вызова более 180 секунд
Условия можно комбинировать при помощи and:
if queue in (SG_50,SG_21) > 1 then priority = 5 and timeWait > 180 then priority = 2
Обязательно в таком случае указывать приоритет для каждого условия `then priority = n`

При симуляции:
Если в выпадающем списке выбирать скилл, то будет отображаться полная статистика именно по этому скиллу, если поле очищать, то будет отображаться краткая сводка по всем скиллам.

Чтобы посмотреть итоговую сводку по симулированным звонкам, необходимо остановить симуляцию, и выбрать необходимый скилл.
